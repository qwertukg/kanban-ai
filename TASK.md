Используя Ktor и Typescript, создай полноценный рабочий продукт, с Бэкэндом и UI, со следующим функционалом:

Система управления задачами (таск трекер) в виде канбана, где каждая задача это чат.
Для каждой колонки (этапа процесса) назначается соответствующий ИИ агент который выполняет работу по задаче в рамках своей роли.
Канбан подключается к GIT репозиторию, движение задачи по доске, создает, модифицирует, и сливает ветку в целевую, по мере продвижения по доске

Имеет следующий функционал:

- создавать доски (backend + UI)
- создавать колонки в досках (backend + UI)
- создавать задачу в колонках (backend + UI)
- создавать ИИ агентов (backend + UI)
- назначать агентам критерии приемки задачи (backend + UI)
- назначать агентам инструкции по его роли (backend + UI)
- назначать ИИ агентов на колонку в доске (backend + UI)
- создавать глобальные инструкции для ИИ Агентов (backend + UI)
- перетаскивать задачи между колонками методом драг-н-дроп

Сущности:
- Доска
- Колонка
- Задача
- Агент
- Настройки (апи ключ, репозиторий итд)

Экраны админки:
- Доски (CRUD + список)
- Колонки (CRUD + список)
- Задачи (CRUD + список)
- Агенты (CRUD + список)
- настройки (поля ввода)

Экраны пользователя:
- один экран с выбранной доской и слева меню со списком досок


Первая и последняя колонка - Open И Closed создаются автоматически, при добавлении в колонку "Open" создается ветка-фича с целевой ветки,
при достижении колонки "Closed" ветка-фича сливается в целевую ветку.
В качестве ИИ используется Codex от OpenAI и подключается через API_KEY.
Каждая задача это чат.

Пример использования Человеком:
1. Пользователь создает доску "Разработка"
2. Создает 4 колонки: Анализ, Разработка, Тестирование, Эксплуатация
3. Создает 4 ИИ агентов: Аналитик, Разработчик, тестировщик, ДевОпс
4. назначает для ИИ агентов инструкции по его роли:
5. назначает для ИИ агентов инструкции по приемке задачи:
6. создает задачу в колонке Open
7. если агент видит задачу в своей колонке, и агент не занят, задача передвигается в его колонку, и агент делает соответствующую запись в задачу и начинает работу в рамках своей роли,
   если критерии приемки не соответствует, то агент возвращает ее в предыдущую колонку и записью о том что нужно доработать в рамках его критериев
8. когда работа закончена, агент делает соответствующую запись в задаче, и задача перемещается в следующую колонку
